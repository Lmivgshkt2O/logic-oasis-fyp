# U7 model artifacts

Only a versioned, evaluated **XGBoost** bundle that declares the U7
`next_attempt_support_needed` target may be promoted to the runtime registry.
The bundle must contain its target/label version, mastery criterion,
`quiz-attempt-features-v2`, exactly `correct_rate` and
`mean_response_time_ms` (or the separately evaluated typed BKT ablation),
dataset version, evaluation-report hash, and manifest hash. It remains a
candidate until the project owner and supervisor approve the evaluated bundle.

`../xgboost_logic_oasis_model.pkl` is a legacy developer artifact using the
old `Weak`/`Moderate`/`Strong` mastery-label path. It is not an active U7/U8
model, must not be registered, and must not be used for FYP1 model claims.

Candidate bundles are generated by `training/evaluate_models.py` only after a
fair student-grouped comparison using seed `20260716`. Synthetic-test bundles
are temporary test evidence only and cannot be promoted. Promotion is explicit;
no quiz completion retrains or activates a model.
